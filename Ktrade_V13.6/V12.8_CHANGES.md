# KTrade — V12.8 Changes (production-resilience audit)

Addresses the three highest-value, contained items from the resilience audit —
the two the audit said "will kill you live" plus the scan-file integrity gap.
Verified against source and covered by a new regression test
(`test_v128_resilience.py`). Larger items (monolith split, full Monte-Carlo /
shadow harness) are scoped separately below — not rushed into safety-critical code.

## Fixed
1. **Dormant VIX/regime sensor now fails conservative (audit Scenario 4).**
   `MARKET.vix` defaulted to a calm 18.0 and only updated when `market_fn` was
   wired, so the VIX circuit breakers never fired by default — a "smart bug" that
   passed every offline test. Now: VIX feed freshness is tracked
   (`vix_updated_at`), and the autonomous **pre-cycle guard refuses new entries**
   when there is no fresh VIX feed within `KTRADE_MAX_VIX_AGE_MINUTES` (default 15)
   — logged as "REGIME STALE — conservative mode". Set
   `KTRADE_REQUIRE_FRESH_VIX=false` to trade blind (not recommended).
   *Design note:* this lives in the autonomous pre-cycle guard, **not** in
   `RiskEngine.evaluate`, so regime policy doesn't couple into the per-trade risk
   primitive (an earlier attempt to put it in `evaluate` broke gate-probing tests
   — the right layer is the loop).

2. **Desync → halt is now wired into the autonomous loop (audit Scenario 5 — the
   highest-severity gap).** Before each cycle, `_broker_truth_guard()` compares the
   broker's real positions against the agent's internal `open_positions`:
   - **Phantom position** (agent thinks it holds a name the broker doesn't) →
     halt new entries, fire the kill switch (cancel orders; flatten only if
     `KTRADE_DESYNC_AUTO_FLATTEN=true`, default off).
   - **Broker unreachable** for `KTRADE_MAX_BROKER_FAILS` consecutive cycles
     (default 3) while holding risk → halt new entries. Conservatively skips the
     cycle whenever broker truth can't be verified. Server-side brackets continue
     to protect open positions.

3. **Atomic scan-file write (audit Scenario 3 — TOCTOU/partial-write).**
   `--score-only` now writes `ktrade_scan_latest.json` via temp + `os.replace`, so
   a concurrent backend read can never observe a half-written file.

## Verification (this build)
- `py_compile` clean. New `test_v128_resilience.py`: **8/8** (skips gracefully
  without deps). Proven via a fake broker: stale-VIX halt, in-sync pass, phantom
  desync → kill fired, broker-unreachable x3 → halt.
- Full offline sweep: **13/13 suites** (the regime guard was deliberately kept out
  of `evaluate`, so the existing gate tests are unaffected — confirmed by reverting
  an earlier evaluate-level attempt that broke 4 suites).
- `scripts/check_release_safety.py`: passes.

## Deferred (bigger efforts — recommend doing as focused, separately-tested work)
- **Per-call broker timeout / circuit-breaker in the hot path.** `await_fill` has a
  timeout; `get_positions`/`get_account` should too, so a *hanging* (not just
  failing) broker call can't stall the loop. The failure-streak guard handles
  failures, not hangs.
- **Mandatory live `market_fn`.** The conservative halt now makes a missing feed
  *safe* (no trades) rather than *dangerous* (trading blind), but wiring a real
  lightweight VIX/SPY poll is still the right next step to actually trade.
- **Scan-file signing.** Atomic write closes the partial-write race; an HMAC would
  also close tampering (like the portfolio snapshot already has).
- **Monolith split + full validation harness** (Monte-Carlo drawdown, shadow-mode
  regime-vs-baseline, multi-day FakeBroker production-spine). These are substantial
  and shouldn't be rushed onto the 2190-line core; worth a dedicated effort.

## Unchanged
- Rotate the keys exposed in the pre-V12 zips.
- Keep `KTRADE_PAPER_ORDER_SUBMISSION=false`, `LIVE_TRADING=false`,
  `KTRADE_MANUAL_ALLOW_DEMO=false` until a real paper round-trip is done.
