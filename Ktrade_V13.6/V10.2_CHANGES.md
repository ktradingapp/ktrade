# KTrade v10.2 — Changes (on top of v10.1)

Adds a bad-tick / decimal-shift guard, motivated by a peer agent's phantom KLAC loss
(full write-up in `reference/KLAC_baddata_finding.md`).

## Files changed

### NEW — `data/price_sanity.py`
`PriceSanityGuard` + module singleton `PRICE_GUARD`. Validates every live price
against a per-symbol reference and rejects decimal-shift errors (price/ref ≈ a power
of 10), implausible single-tick jumps, and non-positive prices.
- `scrub(prices)` — quote-ingestion filter; carries forward last-good on rejection.
- `validate_entry(symbol, price, reference)` — hard no-trade gate.
- `seed_reference(symbol, price)` — optional bootstrap from prior close so the FIRST
  tick can be validated too.

### CHANGED — `data/ktrade_data.py`
- Imports `PRICE_GUARD` (robust to run directory; no-ops if unavailable).
- `PolygonDataFeed.get_snapshot()` now scrubs the snapshot through the guard before
  returning, so a corrupted quote never reaches signal/momentum logic.

### NEW — `data/test_price_sanity.py`
Proves the guard rejects the real KLAC $2300.52 tick (`decimal_shift_x10`), refuses
the entry, still passes a genuine +8% move, and catches a 10x-too-low tick. All pass.

### NEW — `reference/KLAC_baddata_finding.md`
The diagnosis and raw record.

## Recommended follow-up wiring (optional, not auto-applied)
Add a hard gate at the decision stage too, so even a guard miss can't place a trade:

```python
from price_sanity import PRICE_GUARD   # adjust import path to your run dir
gate = PRICE_GUARD.validate_entry(ticker, entry_price, reference=prior_close)
if not gate["ok"]:
    log.warning("Entry blocked — bad price: %s", gate["reason"])
    return  # skip this candidate
```

And seed references each morning from prior close so the first live tick is covered:
`for sym, close in prior_closes.items(): PRICE_GUARD.seed_reference(sym, close)`
