# KTrade — V13.14 (broker wiring, standalone VIX feed, pipeline timeouts)

The latest review was the best yet — it ran all 25 suites and found one genuinely
critical deployment bug plus two real robustness issues. All three fixed; its other
points assessed below.

## 1. CRITICAL FIX: autonomous agent now wires the paper broker
`main()` created `KTradeCEO(market_fn=...)` with NO broker, so `ktrade-agent.service`
scanned and approved but ExecutionAgent returned simulated/no-fill — it was never a
true autonomous paper-trading service. Now, when `KTRADE_PAPER_ORDER_SUBMISSION=true`,
`main()` wires `AlpacaBrokerAdapter` into the CEO (bracket orders, broker-truth sync,
reconcile guard all engage). Default (false) keeps the previous scan-only behaviour.
Fails safe: if the adapter can't load, it logs and runs scan-only.
**Do not run agent autonomous trading and backend auto-trading simultaneously** —
pick one path (documented in .env.template).

## 2. VIX/SPY feed no longer depends on the Flask backend
The v13.13 market_fn imported `broker_adapter` -> `backend.ktrade_alpaca` (Flask), so
regime protection failed wherever dashboard deps were missing (the review reproduced
"No module named 'ktrade_alpaca'"). New `make_standalone_market_fn()` uses
PolygonDataFeed directly for BOTH VIX and SPY — same strict fail-safe (omit on
failure; 5-150 sanity gate). Verified here with Flask hard-blocked: returns {} and
never crashes.

## 3. Validation pipeline: per-test timeout with LOUD reporting
One slow/hanging suite could stall the whole `--validate` run. Each test now gets
`KTRADE_PIPELINE_TEST_TIMEOUT_S` (default 300s) and a timeout is reported as
`<test> TIMEOUT(>Ns)` in the FAIL list instead of freezing. Mechanism verified
(rc=124 path exercised with a deliberate slow child).

## The review's other points — assessed
- **Signal-invalidation exit** (close a long when MACD/trend flips): real gap, and
  correctly labeled "acceptable for paper / future fix". It's a STRATEGY change that
  alters the P&L profile — the broker-side bracket remains the protective exit —
  so it goes on the validate-on-your-data backlog, not in blind.
- **KTRADE_REQUIRE_FRESH_VIX=false in .env.template**: deliberate paper-collection
  choice, already documented; the review's own recommendation (=true once VIX
  verified on VPS) matches the standing plan.
- **Stale bundled scan file**: safe by design (stale-scan rejection blocks it);
  running a fresh `--score-only` before any auto-trade test is already step 4 of the
  agreed paper path.
