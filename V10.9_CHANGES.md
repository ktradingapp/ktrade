# KTrade v10.9 — Changes (on top of v10.8)

Closes the earnings-awareness gap identified by comparing against the peer
(TradeRadar) agent: KTrade can now AVOID opening risk before earnings and EXIT
holdings ahead of a binary earnings event — the "exit ahead of MU earnings"
behavior — using the free Finnhub earnings calendar.

Tests: 22 + 9 + 6 + 18 + 11 + 9(earnings) + price-sanity, all pass (75 total).

## New
- data/earnings_calendar.py — EarningsCalendar: next earnings date per ticker via
  Finnhub /calendar/earnings (free-tier friendly), on-disk cache + TTL, pluggable
  provider for testing. FAILS OPEN: no key / no data -> gate is a no-op (never
  blocks trading on a data outage).
- test_v109_earnings.py — 9 checks (blackout gate + exit-ahead + fail-open).

## Wired
- RiskEngine.evaluate(): BUY is BLOCKED if the ticker reports earnings within
  KTRADE_EARNINGS_BLACKOUT_DAYS (default 3). Reason: "EARNINGS BLACKOUT".
- KTradeCEO.run_cycle(): _earnings_blackout_exits() exits/flags open holdings
  within KTRADE_EARNINGS_EXIT_DAYS (default 2) of earnings; reported in
  results["earnings_exits"]. Actually sells when a broker sell hook is wired,
  otherwise flags the intent.
- CFG: earnings_blackout_days=3, earnings_exit_ahead=true, earnings_exit_days=2
  (all overridable via .env).

## Notes / limits
- Requires FINNHUB_API_KEY for live earnings dates. Without it the gates are
  inert (fail-open) — set the key to activate.
- Exit-ahead sells only execute when the position monitor's broker sell hook is
  wired; in score-only/no-broker mode it flags intent without selling.
- This is event-AVOIDANCE (risk discipline), not event-prediction. It does not
  add insider/analyst/whale/macro feeds (those need paid data and would dilute
  the quant discipline).
