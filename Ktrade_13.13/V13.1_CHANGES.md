# KTrade — V13.1 (copilot ledger-analysis view)

Turns the shadow ledger into an actual verdict instead of eyeballed JSONL. This is
the piece that answers "is the copilot worth trusting?" — pure analysis, no change
to trading behavior.

## New: agent/copilot_analysis.py (stdlib-only, CLI + importable)
Reads logs/ktrade_copilot_ledger.jsonl and produces two report levels.

**Decision-level (works the moment the ledger has rows — no price data needed):**
how often the copilot fired, agreed, disagreed, abstained (LLM-health), the verdict
mix, and the most-vetoed names. Tells you immediately that the layer is alive and
how opinionated it is.

**Outcome-scored (needs forward prices):** the key trick is that **shadow mode takes
every rule-approved BUY regardless of the copilot**, so each ledger row is a real
trade and a copilot SKIP is a *free counterfactual* — it said "don't buy", you
bought anyway, and the forward return says whether the veto would have helped. We
mark each BUY's forward return over a horizon and report:
- agreement quality: avg forward return when the copilot agreed (BUY);
- veto precision: of SKIP/HOLD calls, how many were followed by a loss (correct) vs
  a gain (wrong);
- **NET veto benefit (pct-pts)**: what you'd have gained/lost by skipping the trades
  the copilot disagreed with. **> 0 means following its vetoes would have improved
  paper P&L.** This is the headline number.

Forward returns score the ENTRY decision over N days — the right symmetric basis for
rules-vs-copilot, cleaner than realized exit P&L (contaminated by stop/target timing).

### Usage
```
python agent/copilot_analysis.py                      # decision-level
python agent/copilot_analysis.py --prices prices.csv  # full verdict
        # prices.csv columns: ticker,date,close   (date YYYY-MM-DD)
python agent/copilot_analysis.py --json               # machine-readable
```
The CSV is the no-code path to the verdict (export daily closes from anywhere). To
use a live source instead, pass your own `price_at(ticker, ts_iso, horizon)` into
`build_report()`.

## New: GET /copilot/report (backend)
Thin, guarded endpoint returning the decision-level report as JSON for the
dashboard (`?horizon=N`). Never breaks the backend if the ledger is missing.

## Honest notes
- The verdict is only as good as the forward prices you feed it; with a thin ledger
  (few decisions) it's directional, not conclusive — let weeks of data accrue.
- This measures whether the copilot helps; it still doesn't establish a rule-engine
  edge. It's the instrument that tells you whether to (a) wire a news feed into the
  copilot and/or (b) ever promote it to `active`. Evidence first.
