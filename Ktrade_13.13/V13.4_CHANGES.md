# KTrade — V13.4 (CI workflow + warning cleanup)

Ships the one item from the latest tooling discussion that can be fully verified
here: zero-cost CI. Plus two small cleanups. No trading-logic changes.

## New: .github/workflows/run-tests.yml (GitHub Actions)
Runs the full suite on every push / PR / manual dispatch — the safety net for the
monolith refactor.

**Correction vs the suggested YAML:** the proposed workflow ran `pytest`. That would
FAIL (silently) here — KTrade's suites are standalone scripts that `sys.exit(1)` on
failure, not pytest tests, so pytest collects nothing and passes falsely. This
workflow invokes them correctly (`python <file>`), under a clean isolated
`KTRADE_APP_DATA_DIR` so the EmergencyController SQLite state can't leak between
suites (the v12.6 determinism fix), with the copilot/regime layers forced off.

It also runs `py_compile` on every module and `check_release_safety.py` first. The
install step uses the minimal dep set the suites actually need (numpy, pandas, flask,
flask-cors, requests, pydantic) — the heavy backtesting deps (vectorbt, yfinance,
numba) are omitted because the suites don't import them and they're slow/fragile on
CI. A comment shows how to switch to full `requirements.txt` if wanted.

**Verified here:** the exact CI invocation was reproduced in the sandbox — compile +
release-safety + all 17 suites green. This is the rare item I can prove works rather
than hand over on faith.
