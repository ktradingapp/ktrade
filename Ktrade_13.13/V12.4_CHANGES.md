# KTrade — V12.4 Changes (third-party review fixes)

Applies the V12.3 review. Every item was verified against the actual code first.
No trading-strategy changes — security, robustness, and correctness only.

## Backend
- **Real `/emergency/kill` endpoint** (#2). Admin-token protected. Triggers the
  persistent `EmergencyController` (records kill state, cancels open orders, and
  optionally flattens positions via the broker adapter) and disables the auto
  worker. This is now a *true* kill switch, distinct from `/auto/stop` (which only
  pauses the worker).
- **Single-worker scan hardened** (#4). The `run_polygon_score_only()`
  single-worker branch now wraps each symbol fetch in try/except, so one bad
  ticker or API hiccup can't abort the whole scan (the parallel branch already
  did this).

## Frontend (`KTrade_preview.html`)
- **Admin token on all control actions** (#1). `post()` now sends
  `X-KTrade-Admin-Token`; added `adminToken()` / `setAdminToken()` (sessionStorage)
  and a **⚿ Token** button. Start / Stop / Run-Once previously would have returned
  401/403 because no token was sent — now they authenticate, and a failed call
  shows a clear "set your token" message.
- **Kill button → `/emergency/kill`** (#2) with a second confirm for flatten
  ("kill only" vs "kill + flatten"). The label is now accurate.
- **Scan panel bound to live data** (#3): reads `all.signals` (mapping
  `action`/`label` and `conviction`/`score`), falling back to sample data only
  when empty. Footer corrected to **v12.4**.

## Runtime
- **Stale lock cleanup** (#5). `ProcessLock.acquire()` now checks whether the PID
  in an existing lockfile is still alive (`_pid_is_running`) and reclaims the lock
  if the previous process died — a crash no longer blocks relaunch. Verified:
  dead-PID lock is reclaimed; a live lock still blocks a duplicate.
- **Desktop env precedence** (#6). The supervisor loads env with
  `override=True`, so in local/desktop runtime the app-data `.env` wins — matching
  the documented intent. The normal backend path is unchanged (project `.env`
  still wins there), so the VPS deployment is unaffected.

## Docs / hygiene
- **README** (#7): version labels → v12.4; stale `ktrade_v9` path examples removed.
- **`.env.template`** (#8): `sk-…`-style placeholders replaced with
  `YOUR_OPENAI_API_KEY` / `YOUR_ANTHROPIC_API_KEY` so external secret scanners
  don't false-positive.
