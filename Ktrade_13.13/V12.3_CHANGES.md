# KTrade — V12.3 Changes

Frontend release. New dashboard UI; no backend trading-logic changes.
Built on the verified V12.2 (RuntimeMerged); all V12.1 state-lock fixes retained.

## New
- **Redesigned dashboard** — `frontend/KTrade_preview.html` is now the
  "illuminated folio" design: lapis-lazuli ground, gold-leaf panel rules, jewel
  tones from Indian miniature painting (peacock = gains, sindoor = loss/kill,
  marigold = pending). Display type **Rozha One**, UI type **Mukta**, mono for
  data. Signature: a 24-spoke dharma-chakra that turns when the agent is running
  (respects reduced-motion). Single self-contained HTML file (~24 KB, no build).
  - Live-polls the real backend: `/all` (equity, positions, prices,
    reconciliation, mode), `/auto/status` (run state), `/closed_trades` (journal).
  - Controls POST to `/auto/start`, `/auto/stop`, `/auto/run-once`; the Kill
    Switch confirms then halts via `/auto/stop`.
  - `const API = ""` (relative) → served at `/` by the Flask backend.
  - Renders sample data standalone so it can be previewed without the backend.

## Preserved
- **`frontend/KTrade_preview_legacy_react.html`** — the previous React dashboard,
  kept intact so nothing is lost.

## Feature delta (new design vs. legacy React UI)
The new design covers monitoring + agent control: equity/P&L/risk/positions/
universe, open positions, latest scan (conviction-ranked), performance, trade
journal, start/stop/run-once, kill switch. It does **not yet** replicate three
things that only the legacy UI has — port on request:
- manual buy/sell **order-entry forms**,
- the **AI-advisor chat** (`/ai/advisor`),
- per-ticker **sentiment** lookups.

## Not yet wired to live data (graceful sample fallback)
- The **scan list** and **performance** panels show sample numbers until bound to
  your exact scan / performance JSON shapes. `/all`, `/auto/status`, and
  `/closed_trades` are live.
