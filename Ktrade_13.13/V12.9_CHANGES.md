## What V12.9 does — completes the whipsaw gate
The execution-side gate already existed (the `evaluate()` block that reads a seeded
`intraday_ranges` dict and refuses BUYs above the threshold, wired in the
autonomous loop via `_seed_price_references_from_data`). V12.9 finishes it:

1. **Scanner-level downgrade.** `ConvictionScorer.score()` now computes each name's
   latest-session high-low range, records it on `ConvictionScore.intraday_range_pct`,
   and **downgrades a BUY signal to WATCH** when the range exceeds the threshold —
   so a violent name never reaches the **score-only scan output** as actionable
   (the score-only path does not seed the evaluate-gate's range dict, so without
   this it would still surface whipsaw names as BUY). This mirrors where Kubera
   applies the gate (its `buy_gates` in the scan/enrich phase).

2. **One config source, runtime-tunable.** Both the scanner downgrade and the
   evaluate gate now read the same dynamic helper `_max_intraday_range_pct()`
   (`KTRADE_MAX_INTRADAY_RANGE_PCT`, default 40%, 0 disables). The earlier inline
   read in `evaluate()` and an import-time CFG field were unified — kept dynamic so
   the threshold can be changed at runtime (an import-time CFG field broke that and
   was reverted).

3. **DRY.** The range math is centralized in `_intraday_range_pct(df)`; the seed
   method reuses it instead of duplicating the formula.

## Why this is a real-but-incremental guard (honest framing)
This is a **tail-risk** gate, not an edge. KTrade's curated universe rarely posts a
40%+ intraday range, so it will seldom fire in normal operation. Its value is on
the genuine chaos day — a halt/reopen, a violent earnings gap, a flash-crash-and-
recover, or a single-name glitch — where even a quality name swings wildly and
entry is unsafe regardless of conviction. It complements the price-sanity gate:
price-sanity catches **bad ticks** (decimal-shift data errors, e.g. KLAC $2300 vs
$236); whipsaw catches **real-but-violent** ranges. It does not create return; it
avoids entering on chaos.
