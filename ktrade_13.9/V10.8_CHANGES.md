## PILLAR 1 — Transactional storage (NEW)
- data/ktrade_db.py — KTradeSQLiteStore: WAL mode, busy_timeout, RLock,
  connections CLOSED after each op (the draft leaked handles), upsert risk_state,
  append-only emergency ledger (audit trail). Graceful-degrade on any DB error.
- risk/state_store.py — now writes/reads SQLite when KTRADE_STATE_BACKEND=sqlite
  (default); atomic-JSON fallback preserved. Public API unchanged.
- risk/emergency.py — kill state persisted to the SQLite ledger (audit trail of
  every kill/reset); JSON fallback preserved. Public API unchanged.
- test_v108_sqlite.py — 11 checks incl. 8-thread concurrent-write stress.
