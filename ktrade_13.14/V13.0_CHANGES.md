# KTrade — V13.0 (Milestone 2: copilot advisory layer + autonomous paper trading)

Major version because this adds a new capability class (the long-deferred advisory
research layer), not another hardening patch. Two things ship together.

## 1. Copilot advisory layer (LLM second opinion) — shadow by default
A `CopilotAdvisor` now sits beside the deterministic rule engine. On a
**rule-approved BUY**, it gets an LLM opinion and writes it to an append-only
audit ledger. Three modes (`KTRADE_COPILOT_MODE`):
- **off** — never called. Zero cost, zero behavior change. (Code default.)
- **shadow** — emits a SKIP/BUY opinion + reasoning, logs it, but **never changes
  what trades**. This is where you start: it builds the evidence.
- **active** — a SKIP/HOLD opinion can **veto** a rule BUY. It can *never create* a
  buy the rules didn't approve. Promote here only once the ledger shows it helps.

Safety properties (all covered by `test_v130_copilot.py`, 20/20, with an injected
fake LLM — no network):
- Any LLM error / missing key → **ABSTAIN → fail-open** (defers to rules). LLM
  downtime can never block trading.
- Shadow **never** blocks; active blocks only on SKIP/HOLD; active+BUY does not.
- `should_block()` is the single gating rule, unit-tested directly.
- The ledger records every weighed-in decision and `summary()` tallies agreement /
  disagreement / vetoes — the falsifiable "was the LLM right?" record.

**Honest limit:** KTrade has **no live news feed**. The copilot reasons from
technicals + your held positions (concentration) + regime (VIX) + its own training
knowledge. It is *not* the catalyst engine Kubera has — to get live-news catalyst
vetoes you'd need to add a news source feeding the prompt (separate effort). What
this layer gives you now is the *measurement harness*: run it in shadow, log
disagreements against your paper fills, and find out whether the opinions add value
before trusting them. Scope is rule-approved BUYs (cheap — usually 0-few LLM
calls/cycle), not Kubera's per-candidate ensemble.

## 2. Autonomous paper trading enabled (in `.env.template`)
`.env.template` now sets `KTRADE_PAPER_ORDER_SUBMISSION=true` so the rule agent
places **paper** bracket orders autonomously — giving the shadow layer real
decisions to score against. Code default stays `false` (safe artifact).

**Critical interaction you must know:** the v12.8 regime guard refuses new entries
when the VIX feed is stale. With no `market_fn` wired, paper submission alone places
**zero** trades (every cycle logs "REGIME STALE"). So the template also sets
`KTRADE_REQUIRE_FRESH_VIX=false` — i.e. trade **blind to regime**, which is fine for
paper data-collection but **must be reverted (or a real VIX feed wired) before live**.
Both code defaults remain the safe values; only the template (which you merge onto
your paper VPS) turns trading on.

## How to run it (paper data-collection)
On your VPS `.env`: `KTRADE_PAPER_ORDER_SUBMISSION=true`,
`KTRADE_REQUIRE_FRESH_VIX=false`, `KTRADE_COPILOT_MODE=shadow`, a real
`ANTHROPIC_API_KEY`. Then let it run. Inspect `logs/ktrade_copilot_ledger.jsonl`
(or `CopilotLedger().summary()`) to see where copilot and rules disagreed.

## The path to full automation (unchanged from the plan)
1. **now** — rules auto-trade on paper (this build) + copilot in shadow.
2. **measure** — weeks of ledger data: whose calls were better, especially vetoes.
3. **promote** — flip `KTRADE_COPILOT_MODE=active` so it *can* veto. Evidence first.
