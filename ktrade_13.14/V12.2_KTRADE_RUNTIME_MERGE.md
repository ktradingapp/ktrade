# KTrade V12.2 — KTrade Runtime Merge Changes

This version adds the best runtime/desktop-wrapper patterns into KTrade while keeping the package KTrade-only.

## Added

- `ktrade_runtime/paths.py`
  - OS-aware app-data directory.
  - Runtime `.env` support.
  - Runtime log directory support.
  - Runtime SQLite DB path support.

- `ktrade_runtime/supervisor.py`
  - Starts the backend as a local supervised process.
  - Health-checks `/health` before declaring success.
  - Uses local-only host/port defaults.
  - Optional macOS sleep prevention through `caffeinate`.
  - Graceful shutdown on Ctrl+C.

- `ktrade_runtime/process_lock.py`
  - Prevents accidental duplicate local backend instances.

- `scripts/run_ktrade_local.py`
  - Safe local launcher for KTrade backend.

- `RUN_KTRADE_LOCAL.cmd`
- `run-ktrade-local.sh`
- `docs/KTRADE_LOCAL_RUNTIME.md`

## Updated

- `backend/ktrade_alpaca.py`
  - Loads KTrade runtime env paths.
  - Writes backend logs to runtime log directory when configured.
  - Adds `/runtime/status` endpoint.
  - Uses locked state snapshot in `/health`.
  - Adds locked `append_error()` helper.

- `agent/ktrade_agent_v9.py`
  - Loads runtime env before logging.
  - Supports `KTRADE_LOG_DIR` / `KTRADE_AGENT_LOG`.
  - Version marker updated to `12.2`.

- `data/ktrade_db.py`
  - Uses `KTRADE_DB_PATH` or runtime DB path by default.
  - Keeps legacy `data/ktrade.db` fallback if runtime helpers are unavailable.

## Not added

No external app bridge files, snapshot writers, or external-app-branded docs were added.
This is a KTrade-only runtime improvement.

## Safety posture
