# KTrade — V12.5 Changes (V12.4 review fixes)

Applies the V12.4 review. Each item verified against the actual code; the two
Major items are behavioral risk fixes and are now covered by a regression test
(`test_v125_review_fixes.py`).

## Major
1. **Emergency kill honors a later flatten** (`risk/emergency.py`).
   `trigger()` previously early-returned if the kill switch was already active, so
   "kill only" followed by "kill + flatten" never flattened. Now the
   event/persist happens only on first activation, but broker `cancel_all_orders`
   (and `close_all_positions` when `flatten=True`) run on **every** call.
   Proven: kill(flatten=False) then kill(flatten=True) → cancels=2, closes=1.
2. **Manual BUY never rounds up past approved size** (`backend/ktrade_alpaca.py`).
   `qty = max(1, int(float(gate["qty"])))` could turn a 0.4-share risk approval
   into a 1-share order (more risk than approved). Now it floors the quantity and
   **refuses with 403** if the approved size is < 1 whole share.

## Minor
3. **ProcessLock.release() ownership check** (`ktrade_runtime/process_lock.py`).
   Release now reads the lockfile's PID and only unlinks it if this process owns
   it, so a timing race can't delete a lock held by another live process.
   (Stale dead-PID reclaim from v12.4 retained.)
4. **README wording** — removed "What's New in v9" / "v9 core" framing; the
   `agent/ktrade_agent_v9.py` filename stays (it's the real path), but the package
   version is clearly v12.5. 
5. **Env precedence made explicit** (`ktrade_runtime/paths.py`,
   `supervisor.py`). `_env_candidates()` now yields the runtime/app-data `.env`
   first and the project `.env` second, with `override=False` — so the order
   itself expresses precedence (app-data wins locally; project `.env` still wins
   on the VPS when no app-data `.env` exists). Supervisor no longer needs
   `override=True`.
