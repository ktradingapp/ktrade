## The one manual wire-up left (intentional)

Your agent and the Flask backend run as separate processes, so I did NOT auto-trip
your kill switch from the backend ‚Äî that would be a cross-process side effect on your
live loop. Wire it where YOU want it in the agent loop:

```python
import requests

def check_state_integrity(backend="http://127.0.0.1:5000"):
    try:
        r = requests.get(f"{backend}/reconcile_truth", timeout=5).json()
    except Exception:
        return  # backend unreachable ‚Äî fail safe, don't trade blind; your call
    if r.get("halt"):
        RISK.activate_kill_switch(                       # your RiskEngine instance
            reason=f"position desync ‚Äî {r.get('desyncs')}")
```

Call `check_state_integrity()` once per cycle before placing new entries. This mirrors
your existing DailyLossGuard pattern but guards state integrity instead of P&L.
