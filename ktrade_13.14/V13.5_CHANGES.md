# KTrade — V13.5 (two-tier intraday watchlist)

Implements the genuinely useful part of the hybrid-scanner feedback: a daily scan
publishes a shortlist; intraday scans only consider that shortlist. Deliberately did
NOT implement the suggested hardcoded intraday time-windows — those came from a
~239-trade peer sample and baking them in as rules is curve-fitting; log time-of-day
and validate it on your OWN paper data instead.

## New: agent/intraday_watchlist.py + wiring (OFF by default)
`KTRADE_WATCHLIST_MODE` (off | on, default **off** — changes nothing when off):
- **Daily/pre-market scan** (`KTRADE_SCAN_INTERVAL=1d`) publishes the top-N ranked
  names (`KTRADE_WATCHLIST_SIZE`, default 30) to
  `data/ktrade_intraday_watchlist.json` (atomic temp+rename, UTC timestamp).
- **Intraday scan** (`KTRADE_SCAN_INTERVAL=5m/1m/...`) restricts its universe to that
  fresh shortlist instead of re-scanning the whole ~200-name universe each cycle —
  fewer API calls, less noise, entries only on names the slower daily scan liked.
- **Staleness gate:** a watchlist older than `KTRADE_WATCHLIST_MAX_AGE_HOURS`
  (default 8h) is ignored. If no fresh watchlist exists, the intraday scan takes
  **no new entries** (you must run a daily scan first). Exits/position management are
  unaffected — they run before entry ranking.

This is the architecture the feedback (correctly) preferred over "scan everything
every 60 seconds" — though note KTrade never did the latter: its 60s cadence is only
the 9:30–10:00 open window, and it already acts only on the top-10 ranked names.

## What I did NOT build, and why
- **Hardcoded intraday time-windows** (10:00–11:30 best, etc.): that pattern is from
  ~239 peer trades; hardcoding it imports someone else's overfit. Log time-of-day and
  test it on your own fills first.
- The feedback's config sets the "VIX/regime guard" in the design while defaulting
  `KTRADE_REQUIRE_FRESH_VIX=false` — i.e. the guard is effectively OFF for paper. That's
  fine for data collection but means regime protection is aspirational until you wire a
  real intraday VIX feed. Unchanged here; just flagged honestly.

## Honest framing (unchanged)
This is plumbing for a mode whose edge is still unproven. It changes nothing by
default and is testable here, which is exactly why it was worth building now. But
intraday remains the heaviest, costliest, least-validated mode — validate it LAST,
after the daily backtest and the one Alpaca paper round-trip answer whether any
version of the strategy has an edge. Minute-scanning also still needs a real intraday
data feed (Alpaca/Polygon, not yfinance) to be trustworthy.
