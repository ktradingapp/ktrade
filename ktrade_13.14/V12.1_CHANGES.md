# KTrade — V12.1 Changes (open-item closeout)

Closes the remaining open items from the V12 verification. No new features —
hardening + hygiene only. Built on the verified-clean V12.

## Fixed
1. **#5 — State-lock pass (the substantive one).**
   The two background writers (`refresh_after_order` and `refresh_loop`) were
   mutating multiple `state[...]` fields **outside** `STATE_LOCK`, so a dashboard
   read could catch a half-updated snapshot. Both multi-field updates are now
   wrapped in `STATE_LOCK` (reentrant, so safe). The single-field read routes
   (`/account`, `/positions`, `/prices`, `/orders`, `/reconciliation`) now read
   through the locked `snapshot_state()` instead of touching `state[...]`
   directly. `/all` already used `snapshot_state()`. The startup connect block is
   single-threaded (pre-concurrency) and left as-is.
   Result: state writes are atomic and all reads are consistent.

2. **#6 — README / version.**
   Header updated `v10.9 → v12.1` with a current-version block summarizing the
   v11.1→v12.1 additions and a "score-only first" safe quick-start. (Autonomous
   mode is not advertised as the default start.)

3. **`test_v107_fixes.py` correction.**
   Replaced the deprecated `pd.Timestamp.utcnow()` with `pd.Timestamp.now("UTC")`.
   Note: the two `#6` scan-schema checks require **pydantic** (present in your
   `.venv`). With pydantic installed the suite is **18/18**; without it,
   `validate_scan_dict` falls back to a passthrough and those two checks fail —
   an environment/dependency condition, not a code bug.
